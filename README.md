## Практика 8 — JWT-токены

### Тема
Работа с **JSON Web Token (JWT)**: создание access-токенов, их валидация, защита маршрутов через middleware.

### Что сделано
Доработано приложение из Практики 7: при успешном входе сервер выдаёт JWT access-токен. Добавлен middleware для защиты маршрутов. Реализован защищённый маршрут `GET /api/auth/me`.


**Защищённый маршрут `/api/auth/me`:**
```js
app.get("/api/auth/me", authMiddleware, (req, res) => {
  const user = users.find(u => u.id === req.user.sub);
  if (!user) return res.status(404).json({ error: "User not found" });
  res.json({ id: user.id, username: user.username });
});
```

**Защищённые маршруты товаров** (требуют заголовок `Authorization: Bearer <token>`):
- `GET /api/products/:id`
- `PUT /api/products/:id`
- `DELETE /api/products/:id`

### Технологии
`Node.js` · `Express.js` · `bcrypt` · `jsonwebtoken` · `nanoid`

## Практика 9 — Refresh-токены

### Тема
Реализация механизма **refresh-токенов**: разделение access- и refresh-токенов, ротация токенов, защита от повторного использования.

### Что сделано
Доработано приложение из Практики 8. Теперь при входе выдаётся пара токенов. Добавлен эндпоинт `POST /api/auth/refresh` для бесшовного обновления сессии без повторного ввода пароля.

**Почему два токена:**
- **Access-токен** — короткоживущий (15 мин), передаётся в каждом запросе
- **Refresh-токен** — долгоживущий (7 дней), используется только для получения новой пары токенов


### Технологии
`Node.js` · `Express.js` · `bcrypt` · `jsonwebtoken` · `nanoid`

## Практика 11 — Система ролей и прав доступа (RBAC)

### Тема
Реализация **Role-Based Access Control (RBAC)**: разграничение прав доступа к маршрутам в зависимости от роли пользователя.

### Что сделано
Доработано приложение из Практики 10: добавлена система ролей. Каждый маршрут теперь доступен только пользователям с соответствующей ролью. Фронтенд отображает интерфейс в зависимости от роли авторизованного пользователя.

**Роли в системе:**
- **Гость** — неаутентифицированный пользователь
- **Пользователь** — только просмотр товаров
- **Продавец** — добавление и редактирование товаров
- **Администратор** — права продавца + управление пользователями

**Таблица маршрутов и прав доступа:**

| Маршрут | Метод | Доступ | Описание |
|---------|-------|--------|----------|
| `/api/auth/register` | `POST` | Гость | Регистрация пользователя |
| `/api/auth/login` | `POST` | Гость | Вход в систему |
| `/api/auth/refresh` | `POST` | Гость | Обновление пары токенов |
| `/api/auth/me` | `GET` | Пользователь | Получение текущего пользователя |
| `/api/users` | `GET` | Администратор | Получить список пользователей |
| `/api/users/:id` | `GET` | Администратор | Получить пользователя по ID |
| `/api/users/:id` | `PUT` | Администратор | Обновить информацию пользователя |
| `/api/users/:id` | `DELETE` | Администратор | Заблокировать пользователя |
| `/api/products` | `POST` | Продавец | Создать товар |
| `/api/products` | `GET` | Пользователь | Получить список товаров |
| `/api/products/:id` | `GET` | Пользователь | Получить товар по ID |
| `/api/products/:id` | `PUT` | Продавец | Обновить параметры товара |
| `/api/products/:id` | `DELETE` | Администратор | Удалить товар |


**Возможности клиентской части по ролям:**

- **Пользователь** — просмотр списка товаров, детальной страницы товара
- **Продавец** — всё выше + создание и редактирование товаров
- **Администратор** — всё выше + удаление товаров, просмотр списка пользователей, обновление информации о пользователях, блокировка пользователей

### Технологии
`React.js` · `axios` · `JWT (роли в payload)` · `Express.js` · `bcrypt` · `jsonwebtoken` · `RBAC`
